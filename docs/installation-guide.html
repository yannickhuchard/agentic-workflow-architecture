Installation & Environment Setup

Complete guide to installing AWA SDKs, visualization engines, and setting up the infrastructure for your agentic workflows.

1. SDK Installation

AWA provides native SDKs for the three most common enterprise environments. Choose your stack below to get started.

# Install the core AWA SDK
npm install @awa/sdk

# Optional: Install the visualization layer
npm install @awa/viz
# Basic SDK
pip install awa-sdk

# With Google Cloud Datastore support
pip install "awa-sdk[datastore]"
<!-- Add to your pom.xml -->
<dependency>
    <groupId>io.awa</groupId>
    <artifactId>awa-sdk</artifactId>
    <version>1.2.0</version>
</dependency>

2. Visualization Setup

The awa-viz.js library allows you to render your workflows in any web-based environment. It supports both 2D (ReactFlow) and 3D (Babylon.js) views.

Direct Inclusion (UMD)

For simple projects or documentation portals, include the dependencies via CDN:

<!-- React & ReactDOM -->
<script src="https://unpkg.com/react@18/umd/react.production.min.js"></script>
<script src="https://unpkg.com/react-dom@18/umd/react-dom.production.min.js"></script>

<!-- ReactFlow (2D) -->
<script src="https://cdn.jsdelivr.net/npm/reactflow@11/dist/umd/index.min.js"></script>

<!-- Babylon.js (3D) -->
<script src="https://cdn.babylonjs.com/babylon.js"></script>

<!-- AWA Viz -->
<script src="path/to/awa-viz.js"></script>

3. Infrastructure & Database

AWA is database-agnostic but provides optimized connectors for high-performance systems.

Firebase / Datastore

Recommended for serverless architectures. Use the Google Cloud Datastore client for production scale.

PostgreSQL

Standard relational backend. Use our provided DDL scripts to initialize your tables.

Backend Configuration:

{
  "AWA_DB_MODE": "datastore",
  "AWA_PROJECT_ID": "your-project-id",
  "AWA_METRICS_ENABLED": "true"
}

4. Schema Management

AWA is built on top of high-performance schema definition languages.

  • Protobuf: Binary serialization for high-speed agent communication.
  • Avro: Schema-evolution friendly format for big data lakes.
  • OpenAPI: Industry standard for RESTful API documentation.
