Agentic BPM Guide

Transforming business processes from human-centric to agent-orchestrated value streams.

The Shift to Agentic BPM

Traditional Business Process Management (BPM) focuses on control flow. Agentic BPM focuses on outcome autonomy and shared context.

  • Contextual Awareness: Agents read and mutate a shared context to make informed decisions.
  • Dynamic Routing: Gates are powered by AI logic and queryable state.
  • Hybrid Collaboration: Seamlessly model handovers between human, ai_agent, and application.

Value Stream Mapping (VSM)

For **BPM Managers**, AWA provides a quantitative way to map process efficiency across AI and Human actors.

The AWA VSM Methodology

  1. Identify Actors: Tag every activity with the correct actor_type.
  2. Set Thresholds: Define bottleneck_threshold_ms based on SLA requirements.
  3. Analyze Transfer: Use transfer_type to identify friction points between silos.
  4. Waste Heatmap: Use the 3D visualization to see "Heat" where AI agents are over-processing context.

Security, Governance & Compliance

For **GRC Officers**, AWA offers declarative safety controls.

Least Privilege Context

Explicitly define read, write, or read_write access on every workflow context.

Audit-by-Design

AWA schemas map directly to immutable event logs (Avro), ensuring a perfect audit trail of agent decisions.

Role-Based Boundaries

Link role_id to corporate IAM systems to enforce cross-actor permissions.
