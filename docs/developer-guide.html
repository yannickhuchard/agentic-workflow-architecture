Reference Architecture

For **Enterprise Architects**, AWA provides a blueprint for scalable multi-agent ecosystems.

The Sovereign Actor Pattern

In AWA, every actor (AI or Human) is treated as a sovereign entity with explicit context boundaries. This prevents "context sprawl" and ensures that security permissions are declarative.

  • Ingress Layer: API Gateway mapping OpenAPI 3.1 specs to AWA workflows.
  • Orchestration Layer: Managed via AWA SDKs (TS/Python/Java).
  • Data Plane: Context stored in PostgreSQL or Datastore with shared-state visibility.

Language SDKs

// TS Fluent API
import { workflow } from '@awa/sdk';

const process = workflow('Claims', '1.0.0')
  .activity('Verification', { actor_type: 'ai_agent' })
  .build();
# Python Pydantic Models
from awa import workflow

process = workflow("Claims", "1.0.0")\
    .activity("Verification", actor_type="ai_agent")\
    .build()
// Java Object Model
Workflow process = WorkflowBuilder.workflow("Claims", "1.0.0")
    .activity("Verification", ActorType.AI_AGENT)
    .build();

The Specification Reference

Field Type Required Description
workflow_id string Yes Unique identifier for the process.
actor_type enum Yes One of: human, ai_agent, robot, application.
duration_ms integer No Expected or actual execution time.
transfer_type enum No integration_flow, manual_work, notification.

AI Integration Patterns

AWA is designed to work with modern AI orchestrators and MCP (Model Context Protocol).

Context to Prompt

Map AWA context_id directly to LLM prompt templates or RAG injections.

Agentic Handoffs

Define blackboard sync patterns for multi-agent swarm collaboration.

Tool/MCP Mapping

Map AWA access_rights to available tools in the AI agent's catalog.

Advanced Visualization

The awa-viz.js library handles rendering and analytics.

Bottleneck Highlighting: If duration_ms > bottleneck_threshold_ms, the renderer automatically colors the node/edge in red (2D) or glowing red (3D).
